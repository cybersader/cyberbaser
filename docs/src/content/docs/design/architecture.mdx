## TL;DR

- **Hub-and-spoke.** One central **hub** orchestrates many **vault-spokes**. ([Why](/cyberbaser/getting-started/vision/#the-shape-one-hub-many-spokes).)
- **The spokes are a commodity.** Each wiki = a vault rendered to a static site by some forward renderer (Quartz, Starlight, Obsidian Publish — *swappable*) + the surfaces people read/edit it through. The forward "vault → web" render is solved and free; [Quartz already wins it](/cyberbaser/research/assumptions-and-risks/#tested-quartz-prior-art-audit-2026-06-19). **Never couple the architecture to one renderer.**
- **The hub is the product.** Renderer-agnostic, and the only hard part: the **reverse round-trip** (a web edit → a clean vault file), the **trust + moderation** pipeline, and **federation**.
- **v1** is one hub over one (or a few) vaults. **v2+** is the same hub over a [federation](/cyberbaser/research/source-of-truth/#federation-mixed-privacy-and-role-based-access) of them.

Scroll for the picture, then the two halves, the round-trip hot-spot, a contribution's journey, and the boundaries.

## The two halves

Everything in the system is either a **spoke** (commodity, swappable) or the **hub** (the product). Keeping the line sharp is what stops cyberbaser from quietly becoming "yet another publisher."

| | Spokes (commodity) | The hub (the product) |
|---|---|---|
| **Does** | render vault → web; show + let people touch the content | round-trip web→vault, govern contribution, federate |
| **Examples** | Quartz, Starlight, Obsidian Publish; the rendered page, a web editor, Obsidian, GitHub | the round-trip engine, trust curve, moderation queue, discovery layer |
| **Swappable?** | yes — pick the cheapest/best renderer per spoke | no — this is the moat |
| **Who else has it** | everyone (free) | [no one — Quartz's maintainer explicitly disclaimed it](/cyberbaser/research/assumptions-and-risks/#tested-quartz-prior-art-audit-2026-06-19) |

## The hub (the product)

Three capabilities, all renderer-agnostic:

- **Round-trip** — the load-bearing one. Turning a browser edit back into a **clean, Obsidian-valid vault file** is the reverse direction *no static-site generator can do*. The [fidelity spike](/cyberbaser/research/v1-architecture/#block-markdown-fidelity--the-keystone-rd) showed it's achievable (20/21 of the hardest fixtures). This is also what makes safe in-page WYSIWYG editing possible. See [Translation Layer](/cyberbaser/design/translation-layer/).
- **Trust + moderation** — the [maintainer-controlled trust curve](/cyberbaser/concepts/problem/#contribution-is-a-curve-not-a-wall) (anonymous → IP → email → verified → KYC, configurable) feeding a **moderation queue**. Nothing goes live until reviewed, so identity can be as light as the maintainer wants. Composed from existing tools (Discourse-style trust, moderation queues, contribution bots), not built from scratch.
- **Federation** — one hub orchestrating *many* vault-spokes, with DNS-style discovery so they can reference each other. v2+; see the [SSOT federation analysis](/cyberbaser/research/source-of-truth/#federation-mixed-privacy-and-role-based-access).

The hub needs a small **runtime** (auth handshake, write-back, moderation state) — which is why it's *not* a static site, and why the spoke renderers (which are static) stay strictly separate from it.

## The spokes (commodity)

- **The vault — SSOT.** A plain Obsidian vault (markdown + `.obsidian/`), in git. The durable artifact: it works in vanilla Obsidian with zero cyberbaser tooling, and the hub never writes anything cyberbaser-specific into it. (e.g. `cybersader/cyberbase` — but any vault, any domain.)
- **The forward renderer.** Vault → web HTML. A commodity: adopt the best free one rather than maintain a bespoke chain. The current prototype uses Astro + Starlight; the [Quartz audit](/cyberbaser/research/assumptions-and-risks/#tested-quartz-prior-art-audit-2026-06-19) is an open question on whether to swap.
- **The surfaces.** Where people read and edit: the rendered page (with an in-page edit widget that calls the hub), a web CMS, Obsidian, GitHub. All edit surfaces feed the *same* hub pipeline.

## A contribution's journey

Follow the green loop:

1. A reader spots an error and **edits in place** on the published page (or via the web CMS / Obsidian / GitHub).
2. The edit hits the **hub**: the round-trip engine turns it into a clean `.md` diff; the trust curve decides how much scrutiny it gets; it lands in the **moderation queue**.
3. A maintainer (or trusted auto-rule) **approves** it. The hub commits the clean file to the **vault (SSOT)**.
4. The **spoke renderer** rebuilds the published wiki. The change is live. Loop closed — and the vault is still a pristine Obsidian vault.

## Boundaries — what each part must NOT do

The separations that keep the system honest:

| Part | Must **not** |
|---|---|
| **Vault (SSOT)** | hold build artifacts · depend on cyberbaser plugins to work in Obsidian · let its file layout be dictated by the URL scheme |
| **Hub / round-trip** | apply lossy transforms · be coupled to a specific renderer · hold content state outside the commit it produces |
| **Spoke renderer** | be treated as permanent or unique (it's swappable) · hold any contribution/auth state |
| **Edit surfaces** | require Obsidian to be running · keep content outside the vault · make the maintainer do anything special per edit |

## Open architectural questions

See [Open Questions](/cyberbaser/reference/open-questions/) for the full list; the architecturally-significant ones now:

- **Adopt a commodity renderer or keep the bespoke stack?** The [Quartz audit](/cyberbaser/research/assumptions-and-risks/#tested-quartz-prior-art-audit-2026-06-19) says the forward render is free and better elsewhere — worth a spike before more Starlight investment.
- **Where does the hub runtime live?** It needs auth + write-back + moderation state; the [self-hosted Forgejo + PKCE stack](/cyberbaser/design/reference-architectures/self-hosted-forgejo-auth/) is one answer (and eliminates the OAuth proxy).
- **How does the round-trip resolve assets and block-IDs** when a web edit touches embeds (`![[note#^id]]`)? The one genuinely unsolved Tier-1 item.
- **What's the incremental build story** for a vault past ~1000 pages, and does a federation of vaults change it?
