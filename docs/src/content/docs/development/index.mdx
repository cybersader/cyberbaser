## Local dev

```bash
cd docs
bun install
bun run dev              # http://localhost:4321/cyberbaser/
bun run dev --host       # also binds to 0.0.0.0 (Tailscale / LAN / Docker)
```

Dev server hot-reloads on markdown and component changes. The base path is `/cyberbaser` — remember to include it in test URLs.

## Build

```bash
cd docs
bun run build            # static build to docs/dist/
bun run preview          # serve the built output on :4321
```

The build runs the full translation layer (remark/rehype plugins), Pagefind search indexing, and sitemap generation.

## Tests

Playwright smoke tests verify the site builds, key pages render, and the Nova theme / Starlight plugins are wired correctly.

```bash
cd docs
bun run test:local       # runs Playwright against local build
bun run test:e2e         # headed browser
bun run test:e2e:ui      # Playwright UI mode
```

## Deploy

Currently: dev-only. Deploy is wired to **GitHub Pages** via `.github/workflows/deploy.yml` (it builds with Bun, then uses `actions/upload-pages-artifact` and `actions/deploy-pages`). It's held during Phase R: no public deploy until the content is ready. GitHub Pages is the current target only; the renderer and host are swappable, and self-hosting (for example behind Forgejo) is the preferred path for anything that needs identity. Cloudflare, if used at all, is an edge layer (CDN, WAF, rate-limiting), not the host.

When you're ready to publish:
1. Enable GitHub Pages for the repo (Settings → Pages → Source: GitHub Actions).
2. Push to `main`. The workflow builds the site and deploys it via `actions/deploy-pages`. No extra secrets are required.
