Cyberbaser is the **contributable wiki platform** — Obsidian authoring, GitHub as the single source of truth, zero-git contribution for anyone. The roadmap reflects this: research and architecture come first, features are built on that foundation, and every significant decision is [logged with rationale](/cyberbaser/agent-context/zz-log/).

See the [vault vision mining](/cyberbaser/agent-context/zz-log/2026-04-13-vault-vision-mining/) for the original dream, and the [vault tech stack mining](/cyberbaser/agent-context/zz-log/2026-04-13-vault-tech-stack-mining/) for the 300+ curated links that seed the research below.

:::note[How this roadmap is maintained]
This page is the **living roadmap** — only active and future phases live here. Every significant decision gets a dated entry in the [decision log](/cyberbaser/agent-context/zz-log/). Link roadmap items to their log entries so the *why* is always reachable from the *what*.
:::

## Current phase: Research & Foundations

## Research & Foundations — "Know what we're building before we build it"

The [vault research](/cyberbaser/agent-context/zz-log/2026-04-13-vault-vision-mining/) shows ambitions spanning publishing, contribution, and real-time collaboration — but the project identity, CMS choice, auth model, and collaboration strategy are all still open. This phase resolves them before code.

**Research & spec** (iterative, first-principled):

- [ ] **Cyberbaser identity** — is this a publishing tool, a wiki platform, a collaboration framework, or all three? The [vault vision](/cyberbaser/agent-context/zz-log/2026-04-13-vault-vision-mining/) is more ambitious than the current docs describe (includes live anonymous collab, contributor attribution, plugin execution across environments). Need to decide what's v1 vs what's deferred. Shapes every downstream decision.
- [ ] **[CMS landscape survey](/cyberbaser/agent-context/zz-log/2026-04-13-vault-tech-stack-mining/)** — the vault has 20+ CMS options evaluated at varying depths. Need to narrow to 2-3 finalists: [Decap](https://decapcms.org/docs/open-authoring/) (Open Authoring), [Sveltia](https://github.com/sveltia/sveltia-cms), [Tina](https://tina.io/), [EmDash](https://github.com/emdash-cms/emdash) (Cloudflare-built, Astro-native, AI-agent-first — [announcement](https://blog.cloudflare.com/emdash-wordpress/)), [Pages CMS](https://pagescms.org/), and others from [RESEARCH_SOURCES](/cyberbaser/reference/existing-work/). The CMS IS the zero-git contribution path — wrong choice = wrong contribution UX.
- [ ] **Auth model** — GitHub OAuth only, or multi-provider? The [vault has 20+ auth solutions](/cyberbaser/agent-context/zz-log/2026-04-13-vault-tech-stack-mining/) researched ([Better-Auth](https://www.better-auth.com/), [Clerk](https://clerk.com/), [Ory/Kratos](https://www.ory.sh/kratos/), [SuperTokens](https://supertokens.com/), [Authentik](https://goauthentik.io/) with [Wiki.js integration](https://docs.goauthentik.io/integrations/services/wiki-js/), and more). Auth gates the contribution path — scope decision for v1.
- [ ] **Plugin execution abstraction** — how can Obsidian community plugins (Dataview, linting, attachment management) run outside the desktop app? This was the ["hardest layer"](/cyberbaser/agent-context/zz-log/2026-04-13-vault-vision-mining/) identified in the vault vision. **Major update:** Obsidian now has an [official headless runtime](https://github.com/obsidianmd/obsidian-headless) + [CLI tool](https://obsidian.md/cli) (ships with 1.12, Early Access). If community plugins load in headless/CLI mode, this problem may already be solved. Needs hands-on testing. [obsidianless](https://github.com/lucastraba/obsidianless) enables headless Linux servers. [HN discussion](https://news.ycombinator.com/item?id=47197267).
- [ ] **[Collaboration model](/cyberbaser/agent-context/zz-log/2026-04-13-vault-collab-mining/)** — git-only async, or real-time live editing? The vault vision includes "edit with someone → anonymous secure live editing." The [vault survey](/cyberbaser/agent-context/zz-log/2026-04-13-vault-collab-mining/) found 4 models (file-level P2P, live collab via [Peerdraft](https://www.peerdraft.app/)/[LiveSync](https://forum.obsidian.md/t/self-hosted-livesync-ex-obsidian-livesync-released/26673), git-based, workaround methods). 50+ CRDT/sync links from [vault tech stack mining](/cyberbaser/agent-context/zz-log/2026-04-13-vault-tech-stack-mining/) ([Yjs](https://yjs.dev/), [Automerge](https://automerge.org/), [BlockSuite](https://blocksuite.io/), [ElectricSQL](https://electric-sql.com/)). Preliminary recommendation: defer real-time to v2, focus v1 on the three async paths.
- [ ] **Translation layer deep-dive** — does [astro-loader-obsidian](https://github.com/aitorllj93/astro-loader-obsidian) handle [Tier 1 features](/cyberbaser/design/translation-layer/) (wikilinks, callouts, embeds, math, Mermaid) sufficiently, or do we need custom remark/rehype plugins? Changes Phase 1 scope by 5-10x. Requires hands-on testing with real [cyberbase vault](https://github.com/cybersader/cyberbase) content against the [Obsidian Flavored Markdown spec](https://help.obsidian.md/Editing+and+formatting/Obsidian+Flavored+Markdown). Also evaluate [Markdoc](https://markdoc.dev/) as an alternative authoring format. [starlight-theme-obsidian](https://github.com/Fevol/starlight-theme-obsidian) is worth examining.
- [ ] **Publish site survey** — review 10-15 of the 60+ example Obsidian Publish sites catalogued in the vault (`📁 51 - Cyberbase/Cyberbase Obsidian Publish/`). Extract UX patterns: navigation, search, mobile, backlinks, graph view, contributor attribution, SEO. The reader experience determines whether people actually use the wiki.
- [ ] **Data sync strategy** — how does content flow between Obsidian (local) → GitHub (repo) → Web CMS (edits) → built site? The vault [identified specific failure modes](/cyberbaser/reference/open-questions/): Obsidian Git conflicts from date-modified fields, Windows path length limits, bracket escaping in filenames. Media hosting at scale (Cloudinary + S3/Storj per the [vault tech stack](/cyberbaser/agent-context/zz-log/2026-04-13-vault-tech-stack-mining/)). [Resumable uploads via tus.io](https://tus.io/). Architecture: [current diagram](/cyberbaser/design/architecture/).
- [ ] **SEO baseline** — sitemaps, robots.txt, canonical URLs, OG images, structured data. Which does Starlight handle, which need custom work? The vault has research in `📁 51 - Cyberbase/Obsidian Publish SEO/`. [Stable URLs on Obsidian Publish](https://joschua.io/posts/2023/10/07/stable-publish-urls) is the key reference for permalink design (R08 / [content addressability](/cyberbaser/concepts/primitives/#content-addressable--stable-urls)).
- [ ] **Operational model + sustainability** — where does the sustained work live in cyberbaser? Which layer creates enough value that someone would pay to not build/maintain it? Candidate value centers: translation layer as a library (`@cyberbaser/translate`), contribution widget (SaaS-able "add contributability to any Obsidian site"), plugin execution in CI/headless (unique capability), hosted service ("bring your vault, get a contributable wiki"), enterprise tier (RBAC, audit, teams). See [crosswalker's operational landscape](/crosswalker/concepts/operational-landscape/) for the pattern. Informs license choice and what to protect vs. give away. Intertwined with identity — "what is cyberbaser" and "what's the business" answer together.
- [ ] **Inline editing UX** — what does "click edit on any page and immediately edit the content right there" actually mean technically? [Decap widget](https://decapcms.org/), [TinaCMS inline mode](https://tina.io/), [EmDash's AI-agent-first portable text](https://blog.cloudflare.com/emdash-wordpress/), custom [BlockSuite](https://blocksuite.io/)/[Tiptap](https://tiptap.dev/) editor, [Prose](https://github.com/prose/prose) (GitHub content editor), or something else? This is the [dream feature](/cyberbaser/agent-context/zz-log/2026-04-13-vault-vision-mining/) that closes the reader→contributor loop. Synthesizes CMS choice, auth model, and translation layer decisions.

**Infrastructure:**

- [ ] Bug fixes and polish on current docs site (Starlight + Nova + silver/emerald theme)
- [ ] Expand Playwright smoke tests for new research pages
- [ ] Update [`.claude/` numbered files](/cyberbaser/reference/existing-work/) to reflect decisions as they're made

**Exit criteria** — all must be true before Research & Foundations closes:

- [ ] Identity stated in one sentence in [Problem](/cyberbaser/concepts/problem/)
- [ ] CMS shortlist narrowed to 2-3 with a recommended pick and rationale
- [ ] Auth model chosen with v1 scope boundaries
- [ ] Plugin abstraction tested (Obsidian CLI + community plugins) and scoped (v1 or deferred)
- [ ] Collab model decided: git-only for v1 or real-time with evidence
- [ ] Translation layer evaluated against Tier 1 feature list with real vault content
- [ ] 10+ Publish sites reviewed with extracted patterns
- [ ] Data sync architecture documented with failure modes
- [ ] SEO checklist completed (Starlight-handles vs custom-needed)
- [ ] Inline editing UX chosen with reference implementation or mockup
