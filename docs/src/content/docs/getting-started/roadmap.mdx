## Current phase — **R · Research & Foundations** <span class="cb-pill cb-pill-research">Active</span>

**Goal:** produce substantive content in the first-principles pages (Problem, Ecosystem, Concepts, Prior Art, Existing Work, Vision, Principles) so every subsequent phase can cite a specific section as its justification.

**Exit criteria** — all must be true before Phase R closes:

- [ ] [Problem](/cyberbaser/concepts/problem/) has a one-sentence TL;DR plus articulated pain, audience, and explicit scope boundaries
- [ ] [Ecosystem](/cyberbaser/concepts/ecosystem/) has real paragraphs for every tool surveyed (no TODOs)
- [ ] [Concepts](/cyberbaser/concepts/) has "What / Why / How" filled in for each primitive
- [ ] [Prior Art](/cyberbaser/research/prior-art/) has a "Lesson for cyberbaser" sentence for every entry
- [ ] [Vision](/cyberbaser/getting-started/vision/) rewritten from lived evidence, not first-pass reasoning
- [ ] [Principles](/cyberbaser/getting-started/principles/) has a justified "Why" for each principle, grounded in Ecosystem / Prior Art content
- [ ] Open questions list is at most 5 items (rest have moved to Resolved)

**In scope during Phase R:**
- Reading and writing into the content pages
- Auditing the local Obsidian research vault for prior thinking to port in
- Mining the archived Phase-0 docs in `.workspace/` for relevant material
- Surveying sibling projects (crosswalker, cyberchaste) for patterns worth borrowing
- Minor bug fixes to the parked Phase-0 prototype *only* if a research path requires running the site

**Explicitly out of scope during Phase R:**
- New features in the publishing pipeline
- Refactoring the Astro site, Starlight theme, or test infrastructure
- Alternative SSG evaluation (unless research explicitly calls for it)
- Decap CMS / Open Authoring integration work
- Translation layer implementation
- Any code not directly justified by an already-written principle

## Roadmap operating rules

> [!important] Rule 1 — Roadmap is downstream of principles
> If you find yourself adding a step here that isn't justified by a principle in the Principles page, stop and write the principle first (or decide you don't need the step).

> [!important] Rule 2 — Phase R exits only by the exit criteria above
> No "close enough" shortcuts. The whole point of the reinit was to resist the pull toward implementation before thinking is done.

> [!important] Rule 3 — Short-term focus is the FOCUS.md file, not this page
> `.claude/FOCUS.md` is the dated snapshot of what's active right now. This page is the long-term plan. Don't update both redundantly.
